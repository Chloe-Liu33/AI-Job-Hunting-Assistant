# 本地运行指南

零成本、无需 GPU，3 分钟跑起来。需要 **Python 3.10+**。

## 1. 装依赖

```bash
git clone https://github.com/Chloe-Liu33/Agent-job-search.git
cd Agent-job-search
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

> 想用  Agent 标签 / LlamaIndex / OpenAI-SDK / benchmark，再装一份可选包：
> `pip install -r requirements-extras.txt`（不装也能跑主程序）

## 2. 配 API key 和（可选）Qdrant

### 2.1 先创建 `.env`

```bash
cp .env.example .env
```

### 2.2 打开 `.env`

`.env` 是**隐藏文件**（名字以 `.` 开头），用下面任一方式打开编辑：

```bash
code .env      # VS Code（推荐）
# 或
open -e .env   # macOS 文本编辑器
# 或
nano .env      # 终端里编辑，Ctrl+O 保存、Ctrl+X 退出
```

> VS Code 左侧资源管理器里默认就能看到 `.env`；Finder 里按 `Cmd+Shift+.` 才会显示隐藏文件。

### 2.3 填这些值

| 变量 | 必填? | 填什么 |
|---|---|---|
| `GEMINI_API_KEY` | **必填**（二选一） | https://aistudio.google.com/apikey 拿到的 key |
| `GROQ_API_KEY` | 二选一 | https://console.groq.com/keys 拿到的 key（用它则把这行前的 `#` 去掉） |
| `QDRANT_URL` | 想用云端持久化才填 | 形如 `https://xxx.cloud.qdrant.io:6333`（**末尾带 `:6333`**） |
| `QDRANT_API_KEY` | 同上 | Qdrant 控制台生成的 key |

**格式注意**（最常见的坑）：

- 等号两边**不要空格**：`GEMINI_API_KEY=AbC123`（不是 `GEMINI_API_KEY = AbC123`）。
- 值**不要加引号**。
- 行首有 `#` 表示注释 = 不生效；要启用就**把 `#` 删掉**。
- `GEMINI`/`GROQ` 填**一个**即可，程序自动识别，不用改 `LLM_PROVIDER`。
- 不填 `QDRANT_URL` → 自动用本地文件模式；填了 → 用云端 Qdrant。

### 2.4 检查是否被正确读到

填完跑这条，确认 key/URL 真的读进来了（key 会打码显示）：

```bash
python -c "
import os; from dotenv import load_dotenv; load_dotenv()
mask=lambda v:(v[:6]+'…'+v[-4:]) if v and len(v)>12 else (v or '(空)')
print('GEMINI_API_KEY:', mask(os.getenv('GEMINI_API_KEY')))
print('GROQ_API_KEY  :', mask(os.getenv('GROQ_API_KEY')))
print('QDRANT_URL    :', os.getenv('QDRANT_URL') or '(空)')
print('QDRANT_API_KEY:', mask(os.getenv('QDRANT_API_KEY')))
"
```

- `GEMINI_API_KEY`（或 `GROQ_API_KEY`）不是 `(空)` = LLM 能用。
- 想用云端持久化时，`QDRANT_URL` 应显示完整地址且**结尾是 `:6333`**。
- 如果该填的显示 `(空)`：多半是行首还留着 `#`、等号旁有空格、或没保存文件。

## 3. 启动

```bash
streamlit run app/main.py
```

浏览器会自动打开 `http://localhost:8501`。

## 4. 注册并上传

1. 第一次打开先 **Register** 注册一个账号，再 **Log in** 登录。
   - 每个账号的简历/JD/索引互相隔离，只能看到自己的。
2. 登录后在**左侧边栏上传**：
   - **Add CV**（推荐 PDF，也支持 docx/md/txt）→ 点 **Save CV(s)**
   - **Add JD** → 点 **Save JD(s)**
   - 文件列表里点 🗑 删除自己的；🔒 是共享示例 JD（只读，删不了）。

> **隔离方式**：CV 完全私有，只在你自己的工作区。JD 是**混合**——所有人都能看到
> 共享示例（如 Northern Health、RCH），再加上你自己上传的；同名时你上传的会覆盖
> 共享版。账号密码只存加盐哈希。数据存哪取决于后端（见文末"持久化"）。

## 5. 怎么用

1. 上传时**自动嵌入入库**（第一次会下载 ~80MB 嵌入模型，仅一次）；只有换嵌入模型时才需点 **🔁 Rebuild index**。
2. 多份简历时，用侧边栏 **Active CV** 选这次匹配用哪一份。
3. 各标签页：
   - **📊 Rank JDs** — 哪些岗位最匹配
   - **🔍 Fit analysis** — 单个岗位深度匹配分析（开 **Use RAG** 会基于检索片段并标注引用）
   - **✉️ Cover letter** — 生成求职信
   - **🔎 Semantic search** — 在简历+JD 里语义搜索
   - **🤖 Agent** — 工具调用 agent
   - **📈 LLMOps** — 每次调用的延迟 / token / 成本

## 7. 持久化（数据存哪）

存储后端可切换（见 [app/store.py](app/store.py)）：

- **默认 = 本地文件 + FAISS**：账号/CV/JD/索引都在 `data/`（或 `DATA_DIR` 指定的目录）。本地够用；部署到临时容器会被重启清空。
- **推荐永久存储 = Qdrant**：在 `.env` 里设 `QDRANT_URL`（+ `QDRANT_API_KEY`，免费档见 https://cloud.qdrant.io ），账号 + CV/JD + 向量就都存到外部 Qdrant，跨重启不丢；嵌入仍在本地 CPU 免费算，Qdrant 只存/查向量。没设 `QDRANT_URL` 就自动用本地模式。

```bash
# .env（REST 端口是 6333）
QDRANT_URL=https://你的集群.cloud.qdrant.io:6333
QDRANT_API_KEY=...
# —— 或者不用 Qdrant，只把本地数据放持久盘 ——
# DATA_DIR=/data
```

**配好后先做连接自检**（不动主程序，确认 URL/Key 通）：

```bash
python -c "
import os; from dotenv import load_dotenv; load_dotenv()
from qdrant_client import QdrantClient
c = QdrantClient(url=os.environ['QDRANT_URL'], api_key=os.environ['QDRANT_API_KEY'])
print('现有 collections:', [x.name for x in c.get_collections().collections])
print('✅ Qdrant 连接成功')
"
```

- 打印 `✅ Qdrant 连接成功` 即通（第一次集合为空，正常）。
- timeout/连接错 → 检查 URL 末尾 `:6333`、集群是否 Running。
- 401/403 → key 不对或没权限，重新生成。

启动后看**侧边栏最底部**应显示 `Storage: qdrant`（不是 `file`）。注册账号、上传 CV 后会自动建 `chunks`/`accounts` 两个 collection，重启数据还在 = 成功。

## 8. 部署到 Hugging Face（先本地跑通再推）

**建议顺序：先本地验证（反馈快），再推 HF。** 因为本地和 HF 指向同一个 Qdrant，本地建的账号/CV 在 HF 上也能看到。

1. `git push` 整个仓库到一个 **Streamlit** Space（`.env` 不会被推，已 gitignored）。
2. HF 上**不读 `.env`**，改在 **Space → Settings → Variables and secrets** 填同样三个：
   `GEMINI_API_KEY`、`QDRANT_URL`（带 `:6333`）、`QDRANT_API_KEY`。
3. 容器启动后侧边栏同样显示 `Storage: qdrant`，与本地共用一份数据。

> 想让"本地测试"和"线上正式"数据分开，就再开一个免费 Qdrant 集群，本地 `.env` 用 A、HF Secrets 用 B。现在共用一个即可。

## 常见问题

- **侧边栏报「No API key found」**：`.env` 没填 LLM key，或没在项目根目录启动。
- **Storage 显示 `file` 而非 `qdrant`**：`QDRANT_URL` 没被读到——确认 `.env` 在项目根目录、值没拼错、已重启 app。
- **想停止**：终端按 `Ctrl+C`。
- **中文简历检索不准**：把 `.env` 里 `EMBED_MODEL` 改成 `intfloat/multilingual-e5-small`，再点 Rebuild index。
